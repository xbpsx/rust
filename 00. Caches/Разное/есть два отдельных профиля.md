В Rust (через инструмент **Cargo**) по умолчанию настроены два основных профиля сборки, которые решают разные задачи: 

## 1. Профиль `dev` (Debug)

- **Команда:** `cargo build` или `cargo run`.
- **Где искать:** `target/debug/`.
- **Цель:** Максимально быстрая компиляция, чтобы разработчик не ждал долго после каждого изменения кода.
- **Особенности:**
    - **Минимум оптимизаций** (уровень `0`): код работает значительно медленнее (иногда в 10–100 раз), чем в релизе.
    - **Отладочная информация:** бинарный файл содержит символы для дебаггеров (чтобы можно было ставить точки остановки и видеть названия переменных).
    - **Дополнительные проверки:** включены проверки на переполнение целых чисел и `debug_assert!`.

## 2. Профиль `release`

- **Команда:** `cargo build --release` или `cargo run --release`.
- **Где искать:** `target/release/`.
- **Цель:** Максимальная производительность готовой программы.
- **Особенности:**
    - **Глубокая оптимизация** (уровень `3`): компилятор перестраивает код, чтобы он выполнялся максимально быстро.
    - **Долгая компиляция:** из-за работы оптимизатора сборка занимает больше времени.
    - **Без лишнего веса:** отладочная информация и лишние проверки обычно удаляются, чтобы файл был меньше и быстрее.

## Итог для замеров

Как вы верно заметили, **любые тесты скорости (бенчмарки) нужно проводить только в профиле `--release`**. Тестировать скорость в `debug` бессмысленно, так как там Rust жертвует производительностью ради удобства разработки и скорости сборки.
